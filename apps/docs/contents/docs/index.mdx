## How to use this documentation

Everything under `/docs` is structured so executives, engineers, and operators can each land in the right place without hunting through the repo. Use the sidebar to jump directly into the section that matches your job-to-be-done.

### Orientation & feature narratives

- **Executive Overview** highlights the flagship features, AI initiatives, booking flows, and economic impact. Start here if you need the "why" before the "how".
- **Core Quick Start** (`/docs/getting-started`) provides installation, environment, and first-run tutorials for new contributors.
- **Automation Quick Start** walks through the Hello World workflow so you can exercise the automation framework in minutes.

### Setup & delivery

- **Setup Guides** detail database, caching, deployment, and multi-agent worktree procedures.
- **Integrations** cover dashboard, site, and Hashnode sync including production update runbooks.
- **Worktree Management** explains how we structure branches, worktrees, and multi-agent collaboration.

### Engineering & APIs

- **Developer Guide** houses component architecture, package boundaries, and monorepo conventions.
- **API Reference** contains REST, GraphQL, and authentication specs, plus implementation notes.
- **Apps & Packages** outlines how Site, Dashboard, UI, and shared libraries are composed, with links to the relevant Storybook entries under `/docs/component-library`.

### Operations & governance

- **Scripts Reference** enumerates automation helpers for agents, PR triage, and CI/CD workflows.
- **Releases & Security** centralizes release runbooks, security expectations, and incident follow-ups.
- **Troubleshooting** offers curated fixes for common development, deployment, and performance issues.
